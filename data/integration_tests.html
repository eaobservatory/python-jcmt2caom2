Integration Test Set

The validity of the JSA ingestion software can be tested using the observations in the integration test set, which was selected to exercise many of the observing modes and data properties within the JCMT data. The observations are listed here by the original selection criteria, including both raw and processed data, with the commands needed to ingest each dataset. The current version of the test set has expanded beyond the original because the CAOM-2 software ingests whole observations of raw data and whole recipe instances of processed data. For test purposes it is useful if all the raw data used in a recipe instance has been ingested before the processed data is ingested (technically before the metadata is harvested from Sybase into Postgres), so the recipe instance that processed each dataset in the original test set has been identified and the commands needed to ingest all of the raw and processed data are listed below.

The data reduction system processes each subsystem separately, so observations with multiple subsystems, including almost all SCUBA-2 observations, will be reduced using as many recipe instances as there are subsystems. Note that in ACSIS hybrid mode, the hardware subsystems are configured to overlap in frequency and are reduced together as a single hybrid subsystem with a larger bandwidth.

There is a certain amount of repetitiveness in the test set. Many of these observations were originally intended to test the difference between proprietary and public data, to demonstrate that the proprietary access control system worked. With the passage of time, these observations have become public, so the distinction has been removed from the selection criteria. There is still some value in having two examples of each kind of dataset, so both are retained in the test set.
